# cognilearn-hub

### 项目简介:

个人项目，一个希望实用的综合性学习平台项目，打造一个高效、便捷且个性化的学习笔记，助力学习者轻松实现知识的获取与技能的提升。

### 项目资源管理（初步估计）:

一台云服务器（2C4G，后续升级至4C8G）
一个域名（cognilearn-hub的域名）<1元首年，35一年>
第三方服务：邮件服务，消息推送，文件存储
AI 功能：开源大模型取代调用商业大模型
数据监控：使用免费的日志监控工具
HTTPS = HTTP + SSL（Secure Sockets Layer）/TLS（Transport Layer Security）

### 规范开发流程（初步）：

项目用测试驱动开发与领域驱动开发和CI/CD，首先从master分支开分支到feature分支，在对应领域先写单元测试类（覆盖正常使用场景、边界条件以及异常情况），然后做功能开发，功能开发完成，在测试分支做模块间集成测试,与外部系统集成测试，基本安全漏洞检查，在uat分支类生产环境做集成与测压（省略），最后合并到master分支，用Jenkins做构建部署。

## 1. MVP 核心功能验证（先跑起来，能用就行）

**目标**：快速实现最基本的业务功能，保证核心流程可用。

* **核心功能**：
  * 用户注册/短信登录
    * 技术：Spring Security + JWT
    * 模块：`cognilearn-hub-user-auth`
  * 学习笔记（支持 Markdown 富文本，图文发布）
    * 模块：`cognilearn-hub-note-manage`
  * 学习计划（任务创建、进度标记）
    * 模块：`cognilearn-hub-learn-time-plan`
  * 简单前端界面（推荐 Vue）
    * 实现基础的功能页面

## 2. 模块扩展和微服务架构升级

**目标**：增强系统扩展性，引入智能功能及服务治理能力。

* **扩展功能**：
  * AI 辅助学习（笔记摘要、智能推荐）
    * 模块：`cognilearn-hub-ai-learn-assist`
  * 学习网站聚合（资源管理、整合外部学习网站）
    * 模块：`cognilearn-hub-study-web-agg`
* **技术架构改进**：
  * Nacos 注册中心（服务注册与配置管理）
  * Spring Cloud Gateway 统一 API 网关入口
  * OpenFeign 实现服务间声明式调用

## 3. 完善与运维阶段

**目标**：优化用户体验，增强后台管理能力，提升自动化运维水平。

* **完善内容**：
  * UI 美化（响应式设计、深色模式、交互优化）
  * 用户画像与数据看板（图表可视化展示）
  * 后台管理系统（用户管理、笔记审核等）
  * 日志、性能、异常监控
  * 接入 Jenkins 实现 CI/CD 自动化构建与部署
* **模块**：
  * `cognilearn-hub-user-info`（用户信息管理与画像）
